# Инструменты для проверки и улучшения кода

ruff
- Быстрый линтер и форматер для поиска ошибок и проблем со стилем.
- Использование:
  ruff check .          # Проверить весь проект
  ruff check . --fix    # Автоматически исправить ошибки

black
- Автоформатер кода по стандарту PEP8.
- Использование:
  black --check .      # Проверить форматирование без изменений
  black .              # Отформатировать весь проект

monkeytype
- Сбор типов во время запуска и автоматическая генерация аннотаций.
- Использование:
  monkeytype run your_script.py    # Запуск с сбором типов
  monkeytype list-modules          # Список модулей с данными
  monkeytype apply your_module     # Вставить аннотации в код

no_implicit_optional
- Автоматически исправляет неявное использование Optional в аннотациях.
- Использование:
  no_implicit_optional fix path/to/code.py

# Рекомендации
- Сначала запускай monkeytype run для сбора типов.
- Затем применяй monkeytype apply для вставки аннотаций.
- Проверяй код через ruff и black, чтобы избежать ошибок и поддерживать стиль.


!!!!

Запуск тестов/программы с monkeytype run (по необходимости)
— если ты только что добавил или изменил код, нужно сначала запустить программу
 или тесты с monkeytype run your_script.py, чтобы собрать данные о типах.
— Это делается отдельно и не обязательно при каждом коммите, а когда хочешь
добавить или обновить аннотации типов.
monkeytype run main.py

Чтобы увидеть, для каких функций есть собранные типы:
monkeytype list-modules

Если хочешь посмотреть конкретно для одного модуля в консоли, можно:
monkeytype stub ui

Применение аннотаций через monkeytype apply
— после того, как собраны данные, применяешь аннотации к коду с помощью monkeytype
 apply your_module.
— Это улучшает типизацию и помогает инструментам статической проверки.

Запуск no_implicit_optional fix logic.py
— чтобы автоматически исправить случаи, когда Optional используется неявно в аннотациях,
сделай это после вставки аннотаций.
— Это улучшит читаемость и корректность типизации.

Запуск ruff check . --fix
— этот инструмент найдет ошибки, потенциальные баги и несоответствия стилю, а также
автоматически исправит их.
— Использовать до автоформатирования, чтобы устранить возможные логические или
стилистические проблемы.

Запуск black .
— автоформатер приведет код к единому стилю PEP8.
— Делать в самом конце, чтобы не ломать форматирование, сделанное ruff.
black --check . Проверить
black . Применить

Проверка финального состояния кода
— Запусти ruff check . и black --check . без --fix, чтобы убедиться, что ошибок нет и форматирование правильное.

Коммит
— После всех проверок и исправлений код готов к коммиту.
